Sqoop

Sqoop是什么

sqoop是apache旗下一款“Hadoop和关系数据库服务器之间传送数据”的工具。sqoop通过mr导入导出，底层是只有m没有r的mr任务

导入数据：MySQL，Oracle导入数据到Hadoop的HDFS、HIVE、HBASE等数据存储系统；

导出数据：从Hadoop的文件系统中导出数据到关系数据库

CDH先安装了Sqoop2，以为用法一样，只是结构不一样了，后来发现用法完全不一样，又把sqoop给装了回来，还复习了一波
hadoop1 是我的Mysql安装机器
exercise Mysql中测试的的数据库
test_table Mysql exercise数据库的测试表

手动安装sqoop

这里我是CDH直接安装的，下面安装的步骤是apache版本过程

1、上传解压sqoop-1.4.6.bin__hadoop-2.0.4-alpha.tar包

tar -zxvf sqoop-1.4.6.bin__hadoop-2.0.4-alpha.tar

2、修改etc/profile文件

export JAVA_HOME=/root/Downloads/jdk1.8.0_161
export PATH=$PATH:$JAVA_HOME/bin
export HADOOP_HOME=/root/Downloads/hadoop-2.6.5
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
export ZOOKEEPER_HOME=/root/Downloads/zookeeper-3.4.5
export PATH=$PATH:$ZOOKEEPER_HOME/bin
export HIVE_HOME=/root/Downloads/apache-hive-1.2.0-bin
export PATH=$PATH:$HIVE_HOME/bin
export FLUME_HOME=/root/Downloads/apache-flume-1.6.0-bin
export PATH=$PATH:$FLUME_HOME/bin
export HBASE_HOME=/root/Downloads/hbase-1.2.6
export PATH=$PATH:$HBASE_HOME/bin
export SQOOP_HOME=/root/Downloads/sqoop-1.4.6.bin__hadoop-2.0.4-alpha
export PATH=$PATH:$SQOOP_HOME/bin

source /etc/profile 来刷新配置文件来使生效

3、在sqoop下的lib中添加mysql控制包，是负责sql能够识别的

cd /home/hadoop/hive-1.2.1/lib
cp mysql-connector-java-5.1.24-bin.jar /home/hadoop/sqoop-1.4.6/lib

4、修改conf/sqoop-env.sh

mv sqoop-env-template.sh sqoop-env.sh

export HADOOP_COMMON_HOME=/opt/modules/hadoop
#Set path to where hadoop-*-core.jar is available
export HADOOP_MAPRED_HOME=/opt/modules/hadoop
#Set the path to where bin/hive is available
export HIVE_HOME=/opt/modules/hive
export ZOOKEEPER_HOME=/opt/modules/zookeeper
#Set the path for where zookeper config dir is
export ZOOCFGDIR=/opt/modules/zookeeper
#set the path to where bin/hbase is available
export HBASE_HOME=/opt/modules/hbase

5、测试

sqoop version

底层工作机制

将导入或导出命令翻译成 MapReduce 程序来实现

在翻译出的 MapReduce 中主要是对InputFormat 和

OutputFormat 进行定制

参数选项

选项 含义说明
–append 将数据追加到HDFS上一个已存在的数据集上
–as-avrodatafile 将数据导入到Avro数据文件
–as-sequencefile 将数据导入到SequenceFile
–as-textfile 将数据导入到普通文本文件（默认）
–boundary-query 边界查询，用于创建分片（InputSplit）
–columns <col,col,col…> 从表中导出指定的一组列的数据
—delete-target-dir 如果指定目录存在，则先删除掉
–direct 使用直接导入模式（优化导入速度）
–direct-split-size 分割输入stream的字节大小（在直接导入模式下）
–fetch-size 从数据库中批量读取记录数
–inline-lob-limit 设置内联的LOB对象的大小
-m,–num-mappers 使用n个map任务并行导入数据
-e,–query 导入的查询语句
–split-by 指定按照哪个列去分割数据，不同数据交给不同maptask执行
–table 导入的源表表名
–target-dir 导入HDFS的目标路径
–warehouse-dir HDFS存放表的根路径
–where 指定导出时所使用的查询条件
-z,–compress 启用压缩
–compression-codec 指定Hadoop的codec方式（默认gzip）
–null-string 如果指定列为字符串类型，使用指定字符串替换值为null的该类列的值
–null-non-string 如果指定列为非字符串类型，使用指定字符串替换值为null的该类列的值
这里的--table和 --query两者是可以替换的。在使用query时，可以不使用sql，改成–table的形式
使用query时，必须得在Sql语句后面加上/$CONDITIONS 外面单引号可以不加，但是双引号必须加，双引号在shell中有特殊含义
$CONDITIONS 的作用可以参考以下博客
https://www.cnblogs.com/youngchaolin/p/12271211.html

--table traveling_track \

--fields-terminated-by "\001" \
--lines-terminated-by "\n";

sqoop import \
--connect "jdbc:mysql://hadoop1:3306/exercise?useUnicode=true&characterEncoding=utf-8&useSSL=false" \
--username root \
--password 123456 \
--query "select * from traveling_track where \$CONDITIONS" \
--target-dir /user/hive_traveling_track \
--split-by raw_add_time \
--direct \
--null-string '\\N' \
--null-non-string '\\N'

增量导入数据

增量数据：和昨天比，新增的数据和改变的数据，这些数据和昨天的数据进行滚动合并

1.可以通过–query 来手动限制条件，需要在业务表中有相应的字段，不推荐使用

2.append增量导入数据到Hdfs，然后hive中load加载数据

适用情况：表的数据类似hive的事实表，不会发生改变只有新增数据的表。 

sqoop import \
--connect "jdbc:mysql://hadoop1:3306/exercise?useUnicode=true&characterEncoding=utf-8&useSSL=false" \
--username root \
--password 123456 \
--table traveling_track \
--target-dir /user/temp_traveling_track \
--split-by id \
--m 6  \

--incremental append \
--check-column id \
--last-value 392855

–check-column：用指定字段去检查是否符合增量导入的条件
–incremental：指定增量导入的模式,可选 append | lastmodified
–last-value：跟–check-column 配合使用,上次导入的最后一个值,元数据中所有大于last-value的值都会被导入

–m 使用n个map任务并行导入

–split-by 用于拆分工作单元的表格列，默认只能数字进行切分，默认情况下使用文本切分会报错，可通过添加参数使他不报错：

sqoop import -Dorg.apache.sqoop.splitter.allow_text_splitter=true  \

–as-avrodatafile 将数据导入 Avro 数据文件
–as-sequencefile 将数据导入 SequenceFiles
–as-textfile 以纯文本形式导入数据（默认）

3.lastmodified增量导入数据到Hdfs，然后hive中load加载数据

适用情况：新增数据的同时老数据还会发生改变.

这种增量导入方式不支持直接写入hive！需要自己在hive中load

sqoop import \
--connect "jdbc:mysql://hadoop1:3306/exercise?useUnicode=true&characterEncoding=utf-8&useSSL=false" \
--username root \
--password 123456 \
--table traveling_track \
--target-dir /user/temp_traveling_track \
--split-by id \
-m 4 \

--incremental lastmodified \
--merge-key id \
--check-column raw_add_time \
--last-value "2020-07-31 09:31:51";

--check-column: 必须是timestamp列 时间列（int）
--incremental lastmodified: 设置为最后改动模式，基于时间列的增量导入（将时间列大于等于阈值的所有数据增量导入Hadoop）
--merge-key: 合并列（主键，合并键值相同的记录）必须是唯一主键。导入后的数据如果需要跟存量进行合并，使用此参数。 导入的增量数据会将新旧数据进行合并，不会保留。
--last-value: 所有大于最后一个时间的数据都会被更新

Sqoop导入hdfs的null值处理

数据库表中的null值 经过 sqoop导入hdfs中后 再文件中也显示null

而hive加载数据会认为null是字符串，hive中的null是\N

通过下列参数处理：

–null-string ‘\\N’

–null-string ‘\\N’

(两条\\) 第一条是转义字符

–null-string 如果指定列为字符串类型，使用指定字符串替换值为null的该类列的值
–null-non-string 如果指定列为非字符串类型，使用指定字符串替换值为null的该类列的值

Sqoop导出RDbms的null值处理

–input-null-string 如果指定列为字符串类型，使用指定字符串替换值为null的该类列的值
–input-null-non-string 如果指定列为非字符串类型，使用指定字符串替换值为null的该类列的值
