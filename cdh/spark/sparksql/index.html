SparkSql

什么是SparkSQL

Spark是基于sparkcore的一个上层框架，处理结构化的数据

SparkSQL的4个特点

1.易整合

将sql查询与spark程序无缝混合，可以使用java、scala、python、R等语言的API操作。

2.统一的数据访问

以相同的方式连接到任何数据源。

3.兼容Hive

支持hiveSQL的语法。无缝衔接hive

4.标准的数据连接

可以使用行业标准的JDBC或ODBC连接。

SparkSession

使用 Dataset 或者 Datafram 编写 Spark SQL 应用的时候，第一个要创建的对象就是 SparkSession。

SparkSession创建

1、SparkSession在Spark1.x 版本的创建语法，叫做SQLContext

val conf = new SparkConf().setAppName("sql").("local[*]")
val sc = new SparkContext(conf)
//方法已过期
val sqlSC: SQLContext = new SQLContext(sc)

2、SparkSession在Spark2.0开始全新Spark接口

Builder 是 SparkSession 的构造器。 通过 Builder, 可以添加各种配置。

val session: SparkSession = SparkSession.builder().appName("sql").master("local[*]").getOrCreate()

Builder 是 SparkSession 的构造器。 通过 Builder, 可以添加各种配置。

Builder 的方法如下：

Method
getOrCreate 获取或者新建一个 sparkSession，单例模式
enableHiveSupport 增加支持 hive Support
appName 设置 application 的名字
config 设置各种配置
master local 本地单线程
local[K] 本地多线程（指定K个内核）
local[*] 本地多线程（指定所有可用内核）

DataSet

什么是DataSet

spark1.6以后产生的一种操作sql的对象，但是存在很多和rdd相似的便捷之处，可以使用lambda表达式，可以使用转换类的算子，例如map flatMap filter等等，并且存在一个优化了的执行引擎（spark2.0的优势）

DataSet和DataFrame的不同

DataFrame：只能使用sql风格的查询方式，必须通过sc读取hdfs中的文件，然后将文件处理成列信息，然后才能到达dataFrame中，不能直接用dataframe直接进行数据的格式化，DataFrame没有泛型，他的底层类型是DataSet[Row]，Row不能够自己调用，只能他自己生成。Row在获取值时使用.getAs[type] (字段名称或者是索引)

//创建dataSet的时候，里面能够放入的元素必须是样例类的形势，不能放入row
session.createDataset(Row())

DataSet : DataSet的优点是可以使用sql风格的查询方式也可以使用DSL风格，重点是还可以向RDD调用算子一样进行计算，DataSet在创建完毕后拥有类型(类型推断)，DataSet可以放入各种数据类型，但是没办法改别名，只能通过样例类方式和toDF()方法来修改别名，如果是只是传统sql查询方面和DataFrame没有什么区别

Join的三种算法

https://www.cnblogs.com/0xcafedaddy/p/7614299.html

Hash Join

1、broadcast hash join

适用情景：一张小表，一张大表，小表小到可以放入到内存中（广播的表大小必须小于spark.sql.autoBroadcastJoinThreshold，默认是10M）

广播变量形式的join ，小表广播分发到大表所在的分区节点上，效率最高

broadcast hash join可以分为两步：

1、broadcast阶段：将小表广播分发到大表所在的所有主机。广播算法可以有很多，最简单的是先发给driver，driver再统一分发给所有executor；要不就是基于bittorrete的p2p思路；

2、hash join阶段：在每个executor上执行单机版hash join，小表映射，大表试探；

2、shuffle hash join

适用情景：一张小表，一张大表，但是小表不够小，不能放入到内存中，不适合广播分发

先shuffle流程两张表重新组织分配，hashpartition分区后，一个executor变成一个桶，executor再按照hashcode进行处理数据，每个executor处理自己的，单机形式的

shuffle hash join也可以分为两步：

1、shuffle阶段：分别将两个表按照join key进行分区，将相同join key的记录重分布到同一节点，两张表的数据会被重分布到集群中所有节点。这个过程称为shuffle

2、hash join阶段：每个分区节点上的数据单独执行单机hash join算法。

3、sort merge join

适用情景：两张大表，不能放入到内存中，不适合广播分发

1、shuffle阶段：将两张大表根据join key进行重新分区，两张表数据会分布到整个集群，以便分布式并行处理

2、sort阶段：对单个分区节点的两表数据，分别进行排序

3、merge阶段：对排好序的两张分区表数据执行join操作。join操作很简单，分别遍历两个有序序列，碰到相同join key就merge输出，否则取更小一边

//编写自定义UDAF
class MyUDAF extends UserDefinedAggregateFunction{
  //输入数据类型
  override def inputSchema: StructType = StructType(
    List(
      StructField("id",LongType)
    )
  )
  //缓存结果的类型
  override def bufferSchema: StructType = StructType(
    List(
      StructField("sum",LongType),
      StructField("count",IntegerType)
    )
  )
  //最终返回值类型
  override def dataType: DataType = DoubleType
  //保持一致性的开关 默认给true
  override def deterministic: Boolean = true
  //初始化值，这个值得顺序和上面的设定的值类型保持一致
  override def initialize(buffer: MutableAggregationBuffer): Unit = {
    buffer(0)=1L
    buffer(1)=0
  }
  //每次接受到一个数据的时候要进行的计算操作
  override def update(buffer: MutableAggregationBuffer, input: Row): Unit = {
    buffer(0)=buffer.getLong(0)* input.getAs[Long](0)
    buffer(1) = buffer.getInt(1)+1
  }
  //捏合 每个分区中的结果合并在一起
  override def merge(buffer1: MutableAggregationBuffer, buffer2: Row): Unit = {
    buffer1(0) = buffer1.getLong(0)*buffer2.getLong(0)
    buffer1(1) = buffer1.getInt(1)+buffer2.getInt(1)
  }
  //最终返回值
  override def evaluate(buffer: Row): Any = {
    math.pow(buffer.getLong(0),1.0/buffer.getInt(1))
  }
}

class salayUDAF extends UserDefinedAggregateFunction {
  //输入的值
  override def inputSchema: StructType = StructType {
    List(
      StructField("daycount", IntegerType),
      StructField("salarySum", IntegerType)
    )
  }

  //缓存区的内容
  override def bufferSchema: StructType = StructType {
    List(
      StructField("daycount", IntegerType),
      StructField("salarySum", IntegerType)
    )
  }

  //最终返回类型
  override def dataType: DataType = DoubleType

  //一致性
  override def deterministic: Boolean = true

  //初始值
  override def initialize(buffer: MutableAggregationBuffer): Unit = {
    buffer(0) = 0
    buffer(1) = 0
  }

  //单个分区合并
  override def update(buffer: MutableAggregationBuffer, input: Row): Unit = {
    buffer(0) = buffer.getInt(0) + input.getInt(0)
    buffer(1) = buffer.getInt(1) + input.getInt(0) * input.getInt(1)
  }
  //大合并
  override def merge(buffer1: MutableAggregationBuffer, buffer2: Row): Unit = {
    buffer1(0) = buffer1.getInt(0) + buffer2.getInt(0)
    buffer1(1) = buffer2.getInt(1) + buffer2.getInt(1)
  }
  //最终返回值
  override def evaluate(buffer: Row): Any = {
    buffer.getInt(1) * 1.0 / buffer.getInt(0)
  }
}

数据倾斜解决办法

再Hive中分桶时，根据hashcode进行分桶，如果key的值大量重复，则会造成数据倾斜，一个线程处理过大的任务，可能会让系统宕机

UDAF解决问题

在大量重复数据中，加入随机前缀把数据打散，多个线程计算这个数据，减少压力

import org.apache.spark.sql.{DataFrame, SparkSession}

import scala.util.Random

//解决数据倾斜,随即前缀
object RandomKey {
  def main(args: Array[String]): Unit = {
    val session = SparkSession.builder().master("local[3]").appName("randomkey").getOrCreate()
    import session.implicits._
    //匿名udaf！！！
    //加上随机的前缀
    session.udf.register("randomkey", (t: String, number: Int) => {
      val random = new Random()
      val index = random.nextInt(number)
      index + "_" + t
    })
    val df: DataFrame = session.read.textFile("stu.txt").map(t => {
      val strs: Array[String] = t.split(" ")
      (strs(0), strs(1), strs(2).toInt, strs(3))
    }).toDF("id", "name", "salary", "subject")
    
    df.registerTempTable("stu
    //加入1-3的前缀
    val df1: DataFrame = session.sql("select randomkey(subject,3) subject,salary from stu where subject='wx'" +
      " union " +
      "select subject,salary from stu where subject != 'wx'")
    
    df1.registerTempTable("stu1")
    val df2 = session.sql("select subject,count(*) cnt,sum(salary) sal from stu1 group by subject")
    //匿名udaf！！！
    //把前缀去掉
    session.udf.register("cntkey",(t:String)=>{
      t.substring(t.indexOf("_") + 1)
    })
    df2.registerTempTable("stu2")
    session.sql("select cntkey(subject),sum(cnt),sum(sal) subject from stu2 group by cntkey(subject)").show()

    session.close()
  }
}
