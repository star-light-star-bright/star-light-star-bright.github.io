重写和重载的区别

OverLoad
方法的重载
一个方法名可以对应多个方法，只是他们的形参列表不同,可以返回值不同。

override
方法的重写
子类中隔据需要对基类中继承的方法进行重写
重写方法必须和被重写方法具有相同方法名称、参数列表和返回类型

设计模式

什么是设计模式

  1. 静态方法和属性的经典使用

  2. 设计模式是在大量时间中总结的和理论化之后优选的代码结构、编程风格、以及解决问题的思考方式。

单例模式

在 java.lang.Runtime中就使用到了饿汉式的单例模式.

单利设计模式特点：
　　1、单例类只能有一个实例。
　　2、单例类必须自己创建自己的唯一实例。
　　3、单例类必须给所有其他对象提供这一实例。

final关键字

final最终的

  1. 类不可以被继承，加final
  2. 不希望父类的某个方法被子类覆盖重写时,使用final关键字修饰.
  3. 某个类的属性值不能被修改,使用final
  4. 局部变量不希望更改，使用final

细节

  1. final修饰的属性叫做常量，一般用XXX_XXX_XXX命名（有意义、均大写、长度不宜太长）
  2. final修饰的属性在定义时，必须初始化（构造器，代码块或者定义时都可以），并且不能够再修改。
  3. final修饰的属性是static静态的，则只能在定义时或者静态代码块中赋值。
  4. final类不能继承，可以实例化。
  5. 类不是final类，但是含有final方法，虽然不能重写，但是可以被继承。
  6. 类修饰成final类，就不要再final修饰方法了，因为类不能被继承方法怎么可能还会被重写。
  7. final不能修饰构造器
  8. final和static搭配使用，效率更高，底层做了优化，不会导致类的加载（不会调用）
  9. 包装类包括String 类型都是final修饰的，不能被继承的。

抽象类

是一种模板模式。抽象类为所有子类提供了一个通用模板，子类可以在这个模板基础上进行扩充。
通过抽象类，可以避免子类设计的随意性。（继承的子类必须实现抽象类的方法）

  1. abstract修饰类，抽象类

  2. abstract修饰方法，抽象方法，没有方法体

  3. 价值在于设计，设计好后让子类继承并实现抽象类。在框架和设计模式使用比较多。

细节

有抽象方法的类只能定义抽象类

  1. 抽象类不能被实列化
  2. 抽象类不一定包含抽象方法，抽象类可以没有抽象方法，还可以有实现的方法。
  3. 包含了abstract方法的类必须声明为抽象类。
  4. abstract只能修饰类和方法不能修饰属性和其他。
  5. 抽象类可以 包含非抽象方法、构造器、静态属性等等，构造器不能调用，但可以限制子类构造器的格式（抽象类的本质还是类）
  6. 继承了抽象类，那么所有的抽象方法必须被子类实现。或者子类也声明为抽象类。
  7. 抽象方法不能使用private，static，final来修饰，因为这些关键字都是和重写相违背的。
  8. 抽象方法没有方法体
abstract   抽象方法  （只声明）
public abstract class 类名{
    public abstract void 方法名();
}

接口

  1. 接口不能实例化
  2. 接口中所有的public方法，接口中的抽象方法，可以不用abstract修饰
  3. 一个类实现接口就必须将该接口的所有方法都实现。
  4. 抽象类实现接口，可以不实现接口的方法
  5. 类可以实现同时实现多个接口
  6. 接口中的属性都是 public static final 修饰的。
  7. 接口中属性访问形式：接口名.属性名
  8. 接口不能继承其他类，但是可以继承其他接口。
  9. 接口修饰符只能public和默认，和类的修饰符是一样的。
interface 接口名{
    自己的属性；
    自己的方法；

接口和普通类的区别

接口是普通类单继承机制的一种补充。

子类继承父类，自动拥有父类的全部功能。

子类需要扩展功能，通过实现接口的方式扩充。

  1. 继承主要解决代码的复用性和维护性，实现主要价值在于：设计好各种规范，让其他类去实现这些方法。
  2. 接口比继承更加灵活，继承满足is-a 的关系，接口只需要满足like-a的关系。

接口和普通类的参数冲突

接口因为是静态属性，可以使用接口名.属性调用，父类使用super.属性 调用

内部类

一个类的内部嵌套了另一个类的结构，被嵌套的类成为内部类，嵌套其他类的类成为外部类。内部类最大的特点就是可以访问私有属性，体现类和类之间的包含关系。重点！

位置划分

局部内部类

  1. 定义在外部类的局部位置，如方法中，并且有类名。

  2. 不能添加访问修饰符，地位就是一个局部变量，可以使用final修饰类。本质仍然是个类

  3. 可以直接访问外部类所有属性，包括私有

  4. 作用域：仅在定义它的方法或代码块中。相当于这个方法内的局部变量。

  5. 局部内部类可以直接访问外部类的成员。

  6. 外部类其他类不能访问局部内部类，局部类地位是一个局部变量。

  7. 外部类和局部内部类成员重名时，遵循就近原则，先访问外部类的成员，可以直接使用(外部类.this.成员)去访问（外部类.this.成员 代表外部类，直接this.成员代表局部）

public class LocalInnerClass {
    private int n1 = 100;//私有属性

    private void m1() {//私有方法
        //可以修饰final
        final class test {//方法局部内部类
            public void f1() {
                //调用父类私有属性和方法
                System.out.println(n1);
                m1();
            }
        }
        //方法中创建内部类对象，调用方法
        //作用域只能在这个方法中
        test test = new test();
        test.f1();
    }
}

匿名内部类 重点！！

匿名内部类定义在外部类的局部位置，比如方法中，并且没有类名，同时是一个对象！

  1. 不能添加访问修饰符，是一个局部变量
  2. 在定义它的方法或者代码块中
  3. 可以直接访问外部类所有属性方法，包含私有
  4. 不能添加修饰符，因为他本身就是一个局部变量
  5. 仅仅定义在他的方法或者代码块中
  6. 外部其他类不能使用匿名内部类
  7. 就近原则使用，重名使用外部类.this.属性

成员位置划分

成员内部类

  1. 定义在外部类的成员位置上

  2. 可以添加任意修饰符，他是一个成员

  3. 作用域，在外部类中使用

  4. 可以直接访问外部类成员属性方法

  5. 创建成员内部类调用相关属性和方法

  6. 外部类访问成员内部类

  7. 成员重名时，遵循就近原则，使用外部类.this.属性访问外部类属性

静态内部类

外部类成员位置，使用static修饰

  1. 可以直接访问外部类所有静态成员，不能直接访问非静态成员
  2. 可以添加任意修饰符
  3. 作用域：同其他成员为一个整体。
  4. 静态内部类可以直接访问所有静态方法
  5. 外部类访问静态内部类-》创建对象在访问
  6. 外部其他类访问静态内部类 外部类.静态内部类 （前提满足访问权限）
  7. 外部类和静态内部类重名的时候使用就近原则，访问外部类成员，使用（外部类.成员）去访问

    SPRING("春天","温暖"),SUMMER("夏天","很热");

    private String desc;//描述

    private Season(String name, String desc) {

细节

  1. 当我们使用 enum 关键字开发一个枚举类时，默认会继承 Enum 类, 而且是一个static final 类
  1. 传统的 public static final Season2 SPRING = new Season2(“春天”, “温暖”); 简化成 SPRING(“春天”, “温暖”)， 这里必
    须知道，它调用的是哪个构造器.
  2. 如果使用无参构造器 创建 枚举对象，则实参列表和小括号都可以省略
  3. 当有多个枚举对象时，使用’’，’’间隔，最后有一个分号结尾
  4. 枚举对象必须放在枚举类的行首

注解

常用注解

使用An’notation时在前面增加@符号，并在Annotation当成一个修饰符使用。

最基本的Annotation:

@override：重写父类方法，只能用于方法

@Deprecated：表示某个程序元素过期，不建议使用，新老版本之间的过渡

@SuppressWarrings：抑制编译器警告，主要放在方法或者类上面。里面传入一个数组，值去百度，很多

@Interface：代表此类是一个注解类

@Target：修饰注解的注解，称为元注解

元注解

元注解种类

  1. Retention //指定注解的作用范围，三种 SOURCE,CLASS,RUNTIME
  2. Target // 指定注解可以在哪些地方使用
  3. Documented //指定该注解是否会在 javadoc 体现
  4. Inherited //子类会继承父类注解

@Retention 注解

只能用于修饰一个 Annotation 定义, 用于指定该 Annotation 可以保留多长时间, @Rentention 包含一个 RetentionPolicy
类型的成员变量, 使用 @Rentention 时必须为该 value 成员变量指定值:
@Retention 的三种值

  1. RetentionPolicy.SOURCE: 编译器使用后，直接丢弃这种策略的注释
  2. RetentionPolicy.CLASS: 编译器将把注解记录在 class 文件中. 当运行 Java 程序时, JVM 不会保留注解。 这是默认
    值
  3. RetentionPolicy.RUNTIME:编译器将把注解记录在 class 文件中. 当运行 Java 程序时, JVM 会保留注解. 程序可以
    通过反射获取该注解

@Target

修饰Annotation定义，指定annotation被用于修饰哪些程序元素

@Documented

被该注解修饰的Annotation类将被javadoc工具提取成文档，可以看到该注解。

定义为Documented的注解必须设置Retention为RUNTIME！！

@Inherited

被它修饰的Annotation将具有继承性，如果哪个类使用了被@Inherited修饰的Annotation的类，子类将自动具有该注解
