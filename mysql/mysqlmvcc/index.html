MysqlMVCC

MySQL

MVCC

介绍

全称：Multi-Version Concurrency Control，多版本并发控制。指维护一个数据的多个版本，使读写操作没有冲突，快照读为MySQL实现MVCC提供了一个非阻塞读功能。MVCC具体实现，还需要依赖于数据库记录中的三个隐藏字段、undo log日志、readView

概念

当前读

当前读读取的记录的最新版本，读取还要保证其他并发事务不能修改当前记录，会对读取的记录进行加锁。

对于我们日常操作，select…lock in share mode，select … for update，update，insert，delete（排他锁）都是一种当前锁。

快照读

简单的select(不加锁)就是快照读，读取的是数据的可见版本，可能是历史数据，不加锁，是非阻塞读。

Read Committed：每次select，都生成一个快照读

Repeatable Read：开启事务后第一个select语句才是快照读的地方

Serializable：快照读会退化为当前读。

实现原理

三个隐式字段

DB_TRX_ID：事务id，自增型id

DB_ROLL_PRT：指针，指向上一个版本和undo log 配合使用。

DB_ROW_ID：隐藏主键，某张表没主键才会生成

查看.ibd文件

/var/lib/mysql/itcast/

里面查看表中的隐藏字段

ibd2sdi stu.ibd

undo log

回滚日志，在insert，update，delete的时候便于数据回滚的日志

当insert的时候，产生的undo log 日志只在回滚时需要，在事务提交后，即可立即删除。

而update，delete的时候，产生的时候undo log 日志不仅在回滚时需要，在快照时也需要，不会立即删除。

undo log 版本链

不同事物或者相同事物对同一条记录进行修改，会导致该记录的undolog生成一条记录版本链表，链表的头部是最新的旧记录，链表尾部是最早的旧纪录。

readview

ReadView（读视图）是快照读SQL执行时MVCC提取数据的根据，记录并维护系统当前活跃的事务（未提交）id。

ReadView中包含了四个核心字段

四个核心字段：

readview

Read committed 每一次执行快照读都会生成新的ReadView。

根据undo log版本链的事务id进行判断然后选择对应的数据记录读取。

repeatable read 仅在事务第一次执行快照读时生成ReadView，后续一直复用该ReadView。

判断规则和Read committed 一样。
